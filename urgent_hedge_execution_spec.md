# Urgent Hedge Execution Spec

## Purpose

This spec defines the missing execution tooling for urgent EWY downside hedges.
The goal is to turn a live macro shock into a repeatable workflow that can:

- pick a hedge in under `60s`
- stage a probe in under `90s`
- chase intelligently without manual JSON editing
- switch automatically between `same-session`, `into-close`, and `next-open` modes

The problem from the March 10, 2026 EWY hedge miss was not lack of pricing math.
It was lack of a prebuilt urgent-execution system.

- close-aware / session-aware execution policy


## Product Requirements

### Hard requirements

- Never require ad hoc JSON hand-editing during an urgent event.
- Always emit `probe`, `full`, and `open-ready` order files together.
- Support `BAG` combos as first-class execution plans.
- Be able to chase price without duplicating exposure.
- Detect whether options can trade now.
- Switch behavior if less than `15m` remain to the regular options close.

### Safety requirements

- Do not exceed configured max debit.
- Do not overbuy if an earlier probe is still working.
- Do not chase forever.
- Record all order ids generated by the system.
- Keep a strict distinction between:
  - `pricing candidate`
  - `approved plan`
  - `live working order`


## Shared Types

Create new module:

- `urgent_hedge_types.py`

Use frozen dataclasses for internal logic and JSON-serializable dicts for artifacts.

### Macro scenario types

```python
from dataclasses import dataclass, field
from typing import Literal

@dataclass(frozen=True)
class MacroScenario:
    label: str
    horizon_days: int
    spot_move_pct: float
    vol_shift: float
    probability: float
    notes: str = ""

@dataclass(frozen=True)
class MacroScenarioSet:
    name: str
    symbol: str
    as_of: str
    scenarios: list[MacroScenario]
    thesis: str = ""
```

### Hedge universe types

```python
@dataclass(frozen=True)
class HedgeStructureSpec:
    name: str
    expiry: str
    structure: Literal["put", "put_spread", "call_overwrite", "collar"]
    long_strike: float | None = None
    short_strike: float | None = None
    quantity_hint: int = 0
```

```python
@dataclass(frozen=True)
class HedgeCandidate:
    spec: HedgeStructureSpec
    entry_debit: float
    max_value: float
    scenario_pnl: dict[str, float]
    conditional_downside_coverage_pct: float
    expected_pnl: float
    score: float
```

### Combo quote and execution types

```python
@dataclass(frozen=True)
class ComboLegSpec:
    action: Literal["BUY", "SELL"]
    strike: float
    right: Literal["C", "P"]
    expiry: str
    ratio: int = 1
```

@dataclass(frozen=True)
class ComboQuoteSnapshot:
    symbol: str
    spot: float
    legs: list[ComboLegSpec]
    combo_bid: float
    combo_ask: float
    combo_mid: float
    quote_time: str = ""
    source: str = "ibkr"
```

@dataclass(frozen=True)
class ProbePolicy:
    probe_qty: int
    max_wait_seconds: int
    poll_interval_seconds: int
    escalation_ticks: tuple[int, ...]
    stop_after_partials: bool = False
```

@dataclass(frozen=True)
class ChasePolicy:
    max_rounds: int
    tick_up_per_round: float
    max_debit: float
    convert_to_open_ready_if_closed: bool = True
```

@dataclass(frozen=True)
class HedgeExecutionPlan:
    symbol: str
    thesis: str
    combo: ComboQuoteSnapshot
    target_quantity: int
    probe_policy: ProbePolicy
    chase_policy: ChasePolicy
    probe_file: str
    full_file: str
    open_ready_file: str
```

@dataclass
class LiveOrderState:
    order_ids: list[int] = field(default_factory=list)
    status: str = ""
    filled_qty: int = 0
    remaining_qty: int = 0
    avg_fill_price: float = 0.0
```

Purpose:

- fan out over a fixed list of approved EWY hedge structures
- pull current quotes
- run scenario EV
- rank candidates fast
- emit top `N` candidates and the top execution plan

Universe:

- `Mar weekly put spreads`
- `Apr monthly put spreads`
- optionally `call overwrite + put spread` overlays

Functions:

```python

def rank_candidates(candidates: list[HedgeCandidate]) -> list[HedgeCandidate]: ...
def write_ranked_output(path: str, candidates: list[HedgeCandidate]) -> None: ...

CLI:

  --symbol EWY \
  --budget 35000 \
  --output urgent_hedge_ranked.json

### 7. `run_urgent_hedge.py`

Purpose:

- single command entrypoint
- loads macro file
- ranks hedge universe
- picks top candidate
- builds probe/full/open-ready files
- submits the correct one for current session state

CLI:

```bash
uv run python run_urgent_hedge.py \
  --macro ewy_macro_iran.json \
  --symbol EWY \
  --budget 35000 \
  --mode auto
```

Execution behavior:

- if `open` and more than `15m` to close:
  - run probe loop
- if `open` and less than `15m` to close:
  - skip extended probing
  - submit aggressive full ladder
- if `closed`:
  - submit open-ready `GTC` ladder only


## Parallelization Plan

### What should happen in parallel

Immediately when urgent mode starts:

1. fetch current spot and option quotes for all approved hedge structures
2. start order watcher process
3. price all candidates against current macro scenario set
4. build probe/full/open-ready artifacts for the top `2-3` candidates

This should be one command, not manual sequencing.

### Fast-path policy

- `T-30m` to close: full pricing universe allowed
- `T-15m` to close: only top `3` structures priced
- `T-5m` to close: no more ranking expansion, execute only


## Approved EWY Hedge Universe

Do not fan out into dozens of ad hoc ideas during an emergency.
Keep a small curated set.

### Buy-only

- `Mar weekly 135/120 put spread`
- `Mar weekly 135/125 put spread`
- `Apr monthly 135/125 put spread`
- `Apr monthly 140/130 put spread`

### Financed overlays

- `short Mar weekly 150C + buy Mar weekly 135/120 put spread`
- `short Mar weekly 150C + buy Apr monthly 135/125 put spread`

### Ranking metrics

- scenario EV
- conditional downside coverage
- downside pnl in target scenario
- bounce bleed
- cost as % of liquid cash
- execution quality penalty for wide combo market


## Order Lifecycle Rules

### Probe rules

- probe only once if less than `15m` to close
- probe size default: `5` spreads
- probe wait default: `20s`
- if `0` fill after wait:
  - cancel
  - switch to aggressive full ladder

### Chase rules

- never leave old and new combo ladders working at once
- every chase must:
  - cancel old ids
  - confirm cancellation
  - submit new ids
- stop after `3` chase rounds
- stop if new debit exceeds `max_debit`

### Closed-market rules

- if options session closed:
  - no further live repricing loops
  - write open-ready GTC orders
  - submit them once
  - stop churning
